dotnet CLI
Creating a new solution for a web api with multiple projects via the dotnet command
NOTE: If you are using VS Code and the extension solution explorer you will have the possibility to add projects to solution by right clicking on the solution name.
First you set up a folder with the name of the future project. e.g.: WebAPIv3 cd inside of that project folder and type dotnet new sln, no need to specifiy the name as it will take it from the outer folder name.

Creating the web api where the controllers are contained is done at the same level as the sln in the project folder. Type dotnet new webapi -o name_of_main_project and give it a significative name, usually the same as the folder containing the whole project, aka, WebAPIv3

Now add that newly created project to the solution: dotnet sln add WebAPIv3. Apparently a wwwroot empty folder is added but you can delete that.

To run and test dotnet run --project WebAPIv3, you need to specifiy the main project that runs the entire solution.

Now let's add another project that will contain the models needed for the solution. Start with dotnet new classlib -o WebAPIv3.Models and add it to the solution like earlier, dotnet sln add WebAPIv3.Models
To gain intellisense and to add a reference: dotnet add WebAPIv3 reference WebAPIv3.Models. This tells the main project that it needs to know of another project, in this case, the Models. From here Omnisharp will give us suggestions like Use deleteMe.API.Models when referencing any models contained in the models project.

So, create the project as classlib, add it to the solution and finally add a reference to the main project in the solution.

Running the project. Run and watch. Omnisharp considerations for Linux environment.
